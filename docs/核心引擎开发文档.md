## FFprobeParser 类

### 设计思路

FFprobeParser 是一个单例类，负责封装 ffprobe 命令行工具，用于读取媒体文件的元数据信息。主要功能包括：

- 自动定位对应平台的 ffprobe 二进制文件
- 读取媒体文件全量信息（时长、分辨率、编码等）
- 验证媒体文件合法性
- 获取视频关键帧信息
- 获取音频波形数据

### 类结构

```python
class FFprobeParser:
    _instance: Optional["FFprobeParser"] = None
    _ffprobe_path: Optional[Path] = None
```

### 接口方法

#### 1. `__new__(cls) -> "FFprobeParser"`

**说明**: 获取单例实例

**返回值**: FFprobeParser 单例实例

---

#### 2. `set_ffprobe_path(path: str) -> None`

**说明**: 设置自定义 ffprobe 路径

**参数**:
- `path`: ffprobe 可执行文件的路径

**异常**:
- `FFprobeNotFoundError`: 当指定路径不存在时抛出

**示例**:
```python
parser = FFprobeParser()
parser.set_ffprobe_path("/custom/path/ffprobe")
```

---

#### 3. `get_ffprobe_path() -> Optional[Path]`

**说明**: 获取当前 ffprobe 路径

**返回值**: ffprobe 路径的 Path 对象，如果未找到则返回 None

---

#### 4. `get_media_info(file_path: Union[str, Path]) -> Dict[str, Any]`

**说明**: 获取媒体文件的完整信息

**参数**:
- `file_path`: 媒体文件路径

**返回值**: 包含媒体信息的字典

**返回值结构**:
```python
{
    "file_path": str,              # 文件完整路径
    "file_name": str,              # 文件名
    "file_size": int,              # 文件大小（字节）
    "format_name": str,            # 格式名称
    "format_long_name": str,       # 格式全名
    "duration": float,             # 时长（秒）
    "bit_rate": int,               # 比特率
    "has_video": bool,             # 是否包含视频
    "has_audio": bool,             # 是否包含音频
    "video": {                     # 视频流信息（如果有）
        "codec": str,
        "codec_long_name": str,
        "width": int,
        "height": int,
        "pixel_format": str,
        "frame_rate": float,
        "bit_rate": int,
        "duration": float,
        "rotation": int
    },
    "audio": {                     # 音频流信息（如果有）
        "codec": str,
        "codec_long_name": str,
        "sample_rate": int,
        "channels": int,
        "channel_layout": str,
        "bit_rate": int,
        "duration": float
    }
}
```

**异常**:
- `FileNotFoundError`: 文件不存在
- `FileNotReadableError`: 文件不可读
- `FFprobeNotFoundError`: 未找到 ffprobe
- `FFprobeExecutionError`: ffprobe 执行失败
- `ParseMediaInfoError`: 解析媒体信息失败

**示例**:
```python
parser = FFprobeParser()
try:
    info = parser.get_media_info("video.mp4")
    print(f"时长: {info['duration']}秒")
    print(f"分辨率: {info['video']['width']}x{info['video']['height']}")
except Exception as e:
    print(f"错误: {e}")
```

---

#### 5. `is_valid_media_file(file_path: Union[str, Path]) -> bool`

**说明**: 检查文件是否为有效的媒体文件

**参数**:
- `file_path`: 文件路径

**返回值**: True 表示有效媒体文件，False 表示无效

**示例**:
```python
parser = FFprobeParser()
if parser.is_valid_media_file("video.mp4"):
    print("有效的媒体文件")
else:
    print("无效的文件")
```

---

#### 6. `get_video_keyframes(file_path: Union[str, Path]) -> List[Dict[str, Any]]`

**说明**: 获取视频的关键帧信息

**参数**:
- `file_path`: 视频文件路径

**返回值**: 关键帧列表，每个元素包含：
- `timestamp`: 时间戳（秒）
- `pict_type`: 帧类型

**示例**:
```python
parser = FFprobeParser()
keyframes = parser.get_video_keyframes("video.mp4")
for kf in keyframes:
    print(f"关键帧在 {kf['timestamp']}秒")
```

---

#### 7. `get_audio_waveform_data(file_path: Union[str, Path], samples: int = 1000) -> List[float]`

**说明**: 获取音频波形数据

**参数**:
- `file_path`: 音频文件路径
- `samples`: 采样点数，默认 1000

**返回值**: 波形数据列表

**示例**:
```python
parser = FFprobeParser()
waveform = parser.get_audio_waveform_data("audio.mp3", samples=500)
```

## FFmpegEngine 类

### 设计思路

FFmpegEngine 是一个单例类，负责封装 FFmpeg 命令行工具的执行。主要功能包括：

- 自动定位对应平台的 ffmpeg 二进制文件
- 安全的 FFmpeg 命令构建（列表传参，防止注入）
- 同步执行 FFmpeg 命令
- 基于 QThread 的异步执行（支持进度回调、取消）
- FFmpeg 输出日志实时解析和进度计算

### 类结构

```python
class FFmpegEngine:
    _instance: Optional["FFmpegEngine"] = None
    _ffmpeg_path: Optional[Path] = None
```

### 接口方法

#### 1. `__new__(cls) -> "FFmpegEngine"`

**说明**: 获取单例实例

**返回值**: FFmpegEngine 单例实例

---

#### 2. `set_ffmpeg_path(path: str) -> None`

**说明**: 设置自定义 ffmpeg 路径

**参数**:
- `path`: ffmpeg 可执行文件的路径

**异常**:
- `FFmpegNotFoundError`: 当指定路径不存在时抛出

**示例**:
```python
engine = FFmpegEngine()
engine.set_ffmpeg_path("/custom/path/ffmpeg")
```

---

#### 3. `get_ffmpeg_path() -> Optional[Path]`

**说明**: 获取当前 ffmpeg 路径

**返回值**: ffmpeg 路径的 Path 对象，如果未找到则返回 None

---

#### 4. `build_command(*args: str) -> List[str]`

**说明**: 构建安全的 FFmpeg 命令

**参数**:
- `*args`: FFmpeg 命令参数（不包括 ffmpeg 本身和 -y）

**返回值**: 完整的命令列表

**安全特性**:
- 自动添加 `-y` 参数覆盖输出文件
- 采用列表传参模式，完全杜绝命令注入风险

**示例**:
```python
engine = FFmpegEngine()
cmd = engine.build_command("-i", "input.mp4", "-c:v", "libx264", "output.mp4")
# cmd = ["/path/to/ffmpeg", "-y", "-i", "input.mp4", "-c:v", "libx264", "output.mp4"]
```

---

#### 5. `execute_sync(command: List[str]) -> Dict[str, Any]`

**说明**: 同步执行 FFmpeg 命令

**参数**:
- `command`: 完整的 FFmpeg 命令列表（必须以 ffmpeg 路径开头）

**返回值**: 执行结果字典
```python
{
    "success": bool,       # 是否成功
    "returncode": int,     # 退出码
    "stdout": str,         # 标准输出
    "stderr": str          # 标准错误
}
```

**异常**:
- `InvalidFFmpegCommandError`: 命令格式无效
- `FFmpegNotFoundError`: 未找到 ffmpeg
- `FFmpegExecutionError`: 执行失败

**示例**:
```python
engine = FFmpegEngine()
cmd = engine.build_command("-i", "input.mp4", "output.mp4")
try:
    result = engine.execute_sync(cmd)
    if result["success"]:
        print("转换成功！")
except FFmpegExecutionError as e:
    print(f"转换失败: {e}")
    print(f"错误详情: {e.stderr}")
```

---

#### 6. `execute_async(command: List[str], duration: float = 0, progress_callback=None, status_callback=None, log_callback=None) -> FFmpegAsyncTask`

**说明**: 异步执行 FFmpeg 命令

**参数**:
- `command`: 完整的 FFmpeg 命令列表
- `duration`: 媒体总时长（秒），用于计算进度
- `progress_callback`: 进度回调函数 `(progress: float, message: str) -> None`
- `status_callback`: 状态回调函数 `(status: str) -> None`
- `log_callback`: 日志回调函数 `(log: str) -> None`

**返回值**: FFmpegAsyncTask 对象

**示例**:
```python
engine = FFmpegEngine()
cmd = engine.build_command("-i", "input.mp4", "output.mp4")

def on_progress(progress, message):
    print(f"进度: {progress:.1f}% - {message}")

def on_status(status):
    print(f"状态: {status}")

def on_log(log):
    print(f"日志: {log}")

task = engine.execute_async(
    cmd,
    duration=60.0,
    progress_callback=on_progress,
    status_callback=on_status,
    log_callback=on_log
)

task.start()

# 如需取消
# task.cancel()
```

---

### FFmpegAsyncTask 类

FFmpegAsyncTask 继承自 QThread，用于异步执行 FFmpeg 命令。

#### 信号

- `progress_updated(float, str)`: 进度更新信号
- `status_updated(str)`: 状态更新信号
- `log_updated(str)`: 日志更新信号
- `finished(bool, str)`: 完成信号

#### 方法

##### `cancel() -> None`

**说明**: 取消正在执行的任务

**示例**:
```python
task.cancel()
```

##### `is_cancelled() -> bool`

**说明**: 检查任务是否已取消

**返回值**: True 表示已取消

## TaskQueue 与 MediaTask 类

### 设计思路

TaskQueue 是一个线程安全的任务队列管理器，MediaTask 表示单个媒体处理任务。主要功能包括：

- 线程安全的任务队列管理
- 任务生命周期管理（6种状态）
- 多任务串行执行
- 独立的任务进度、日志、结果管理
- 基于 Qt 信号的状态变更通知

### TaskStatus 枚举

任务状态枚举：

```python
class TaskStatus(Enum):
    PENDING = "pending"      # 待执行
    RUNNING = "running"      # 执行中
    PAUSED = "paused"        # 已暂停
    COMPLETED = "completed"  # 已完成
    FAILED = "failed"        # 已失败
    CANCELLED = "cancelled"  # 已取消
```

### TaskQueueSignals 信号

TaskQueue 包含以下 Qt 信号用于通知 GUI 更新：

- `task_added(str)`: 任务已添加（参数：task_id）
- `task_removed(str)`: 任务已移除（参数：task_id）
- `task_status_changed(str, str)`: 任务状态变更（参数：task_id, status）
- `task_progress_changed(str, float, str)`: 任务进度变更（参数：task_id, progress, message）
- `task_log_added(str, str)`: 任务日志添加（参数：task_id, log）
- `queue_empty()`: 队列为空
- `queue_paused()`: 队列已暂停
- `queue_resumed()`: 队列已恢复

**使用示例**:
```python
queue = TaskQueue()

# 连接信号
queue.signals.task_status_changed.connect(
    lambda task_id, status: print(f"任务 {task_id} 状态变为 {status}")
)
queue.signals.task_progress_changed.connect(
    lambda task_id, progress, msg: print(f"任务 {task_id} 进度: {progress}%")
)
```

## 完整使用示例

### 示例 1：读取媒体信息

```python
from src.core.ffprobe_parser import FFprobeParser

parser = FFprobeParser()
try:
    info = parser.get_media_info("video.mp4")
    print(f"文件名: {info['file_name']}")
    print(f"时长: {info['duration']}秒")
    print(f"大小: {info['file_size']}字节")
    if info['has_video']:
        print(f"视频: {info['video']['codec']} {info['video']['width']}x{info['video']['height']}")
    if info['has_audio']:
        print(f"音频: {info['audio']['codec']} {info['audio']['channels']}声道")
except Exception as e:
    print(f"错误: {e}")
```

### 示例 2：同步执行 FFmpeg 命令

```python
from src.core.ffmpeg_engine import FFmpegEngine

engine = FFmpegEngine()
cmd = engine.build_command(
    "-i", "input.mp4",
    "-c:v", "libx264",
    "-crf", "23",
    "-c:a", "aac",
    "output.mp4"
)

try:
    result = engine.execute_sync(cmd)
    if result['success']:
        print("转换成功！")
except Exception as e:
    print(f"转换失败: {e}")
```

### 示例 3：异步执行 FFmpeg 命令 + 任务队列

```python
from src.core.ffmpeg_engine import FFmpegEngine
from src.core.task_queue import TaskQueue, MediaTask, TaskStatus

# 初始化
engine = FFmpegEngine()
queue = TaskQueue()

# 连接信号
def on_task_status(task_id, status):
    print(f"任务 {task_id[:8]}: {status}")

def on_task_progress(task_id, progress, msg):
    print(f"任务 {task_id[:8]}: {progress:.1f}% - {msg}")

queue.signals.task_status_changed.connect(on_task_status)
queue.signals.task_progress_changed.connect(on_task_progress)

# 创建任务
task = MediaTask(
    "convert",
    {"input": "input.mp4", "output": "output.mp4"},
    "视频转换"
)

# 添加到队列
task_id = queue.add_task(task)
print(f"任务已添加: {task_id}")
```

## 注意事项

1. **线程安全**: TaskQueue 内部使用了 `RLock` 保证线程安全，多线程环境下可放心使用
2. **命令注入防护**: FFmpegEngine 的 `build_command` 和执行方法都采用列表传参，完全杜绝命令注入风险
3. **单例模式**: 三个核心类都采用单例模式，确保全局只有一个实例
4. **跨平台兼容**: 所有路径处理使用 `pathlib`，自动适配 Windows/macOS/Linux
5. **异常处理**: 所有方法都有完善的异常处理，抛出项目定义的自定义异常
